# 研修課題解答例：資産の保管場所管理機能の追加

## 1. データベースの変更

テーブル定義と初期データを修正し、保管場所（location）を扱えるようにします。

### 1.1 テーブル定義 (src/main/resources/schema.sql)

`assets` テーブルに `location` カラムを追加します。

```sql
CREATE TABLE assets (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100),
    status VARCHAR(20),
    location VARCHAR(100) -- ★追加
);
```

### 1.2 初期データ (src/main/resources/data.sql)

`INSERT` 文に `location` の値を追加します。


## 2. バックエンドの変更 (Java)

### 2.1 Entityクラス (src/main/java/com/example/demo/entity/Asset.java)

`Asset` クラスにフィールドを追加します。Lombokを使用している場合、Getter/Setterは自動生成されるため、フィールド定義のみで完了します。

```java
package com.example.demo.entity;

import jakarta.persistence.*;
import lombok.Data;

@Entity
@Data
@Table(name = "assets")
public class Asset {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    private String status;

    private String location; // ★追加
}
```

※ `AssetController` などの修正は不要です。Spring MVCのデータバインディング機能により、HTMLのフォーム名とEntityのフィールド名が一致していれば（`name="location"`）、自動的に値が格納されます。

## 3. フロントエンドの変更 (Thymeleaf)

### 3.1 資産一覧画面 (src/main/resources/templates/assets/list.html)

一覧表に「保管場所」の列を追加します。

```html
<!-- ヘッダー部分 -->
<thead>
    <tr>
        <th>ID</th>
        <th>資産名</th>
        <th>保管場所</th> <!-- ★追加 -->
        <th>ステータス</th>
    </tr>
</thead>
<tbody>
    <tr th:each="asset : ${assets}">
        <td th:text="${asset.id}">1</td>
        <td th:text="${asset.name}">MacBook</td>
        <td th:text="${asset.location}">東京本社</td> <!-- ★追加 -->
        <td th:text="${asset.status}">AVAILABLE</td>
    </tr>
</tbody>
```

### 3.2 資産登録画面 (src/main/resources/templates/assets/add.html)

入力フォームに保管場所のテキストボックスを追加します。

```html
<form th:action="@{/assets/add}" method="post">
    <div>
        <label>資産名：</label>
        <input type="text" name="name" required />
    </div>
    
    <!-- ★追加開始 -->
    <div>
        <label>保管場所：</label>
        <input type="text" name="location" placeholder="例: 東京本社" />
    </div>
    <!-- ★追加終了 -->

    <button type="submit">登録</button>
</form>
```
