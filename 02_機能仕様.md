# 機能仕様書

## 1. 認証機能

### 1.1 ログイン
- **概要**: システム利用のためのユーザー認証を行う。
- **入力項目**:
  - ユーザー名
  - パスワード
- **処理フロー**:
  1. ユーザー名とパスワードがDB上の `users` テーブルと一致するか検証する。
  2. 一致した場合、セッションを生成し、メニュー画面へ遷移する。
  3. 不一致の場合、ログイン画面に戻りエラーメッセージを表示する。

### 1.2 ログアウト
- **概要**: システムからログアウトし、セッションを破棄する。
- **処理フロー**:
  1. セッションを無効化する。
  2. ログイン画面へリダイレクトする。

### 1.3 アクセス制御
- **概要**: 未認証ユーザーによる不正アクセスを防止する。
- **仕様**:
  - ログイン画面以外のURLに未ログイン状態でアクセスした場合、ログイン画面へ強制リダイレクトする。

## 2. 資産管理機能

### 2.1 資産一覧表示
- **概要**: 管理対象の資産情報を一覧表示する。
- **表示項目**:
  - 資産ID
  - 資産名
  - ステータス（AVAILABLE: 利用可能 / LOANED: 貸出中）

### 2.2 資産登録
- **概要**: 新規資産をシステムに登録する。
- **入力項目**:
  - 資産名
- **初期値**:
  - ステータスは自動的に `AVAILABLE` で登録される。

## 3. 貸出管理機能

### 3.1 貸出登録
- **概要**: 特定のユーザーへ資産を貸し出す。
- **入力項目**:
  - 貸出対象資産（プルダウン選択）
  - 借用ユーザー（プルダウン選択）
  - 貸出日
  - 貸出予定期間（日数）
- **制約事項**:
  - ステータスが `LOANED` の資産は選択できない（または貸出処理時にエラーとする）。
- **データ更新**:
  - `loans` テーブルにレコードを追加する。
  - 対象資産の `status` を `LOANED` に更新する。

### 3.2 返却処理
- **概要**: 貸出中の資産を返却済みにする。
- **処理フロー**:
  1. 貸出リストから対象の貸出を選択し、返却処理を実行する。
  2. 対象資産の `status` を `AVAILABLE` に更新する。
  3. 貸出レコードを削除（または返却済みステータスへ更新）する。

## 4. ユーザー管理機能

### 4.1 ユーザー一覧表示
- **概要**: 全ユーザーの情報を表示する。
- **表示項目**:
  - ID、名前、権限（Role）

### 4.2 ユーザー削除
- **概要**: ユーザー情報を削除する。
- **制約事項**:
  - **貸出中（未返却）の資産を持つユーザーは削除できない**。
  - 削除を試みた場合、エラーメッセージを表示して処理を中断する。
- **処理フロー**:
  1. `loans` テーブルを参照し、対象ユーザーの未返却データが存在するか確認する。
  2. 存在しない場合のみ、`users` テーブルからレコードを物理削除する。
